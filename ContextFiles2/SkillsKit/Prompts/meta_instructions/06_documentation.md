# Documentation and Hint Generation Tactics

Your goal is to create skill files that can be reused across projects. These tactics will help you decide what to write and how to write it:

- Create a new skill file whenever you identify a tactic or pattern that others can apply in a different system. Group related tactics together; avoid mixing unrelated concepts.
- Use imperative language. Write instructions as if you are guiding another engineer: “Locate the startup file”, “Search for controllers”, “Ignore test projects”.
- Keep the content generic. Remove business names, domain specific terms and other project specific identifiers. Use placeholders or describe the concept abstractly.
- Provide short examples or code snippets when they clarify a tactic. Generic examples help others recognise patterns in their own codebase.
- Avoid duplicating information across skills. If multiple skills refer to the same tactic, link back to a single source or consolidate them.
- At the end of each skill file, include hints for further exploration. Suggest what the next step might be if the reader still has questions.
