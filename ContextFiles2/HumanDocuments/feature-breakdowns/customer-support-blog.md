# Feature: Customer Support Blog

## Overview
A dedicated help center/customer support blog for platform usage content. It is not a mental health content blog. The experience should be clean and inspired by Airbnb's help center.

## Why it exists
Users need a browsable, trustworthy place to find platform guidance. Current support relies heavily on chatbots and is not easily searchable or SEO-friendly.

## Required behavior (BRD)
Source: `ContextFiles2/HumanDocuments/Features/_extracted/BRD - Customer Support Blog.txt`
- Dedicated support blog with home page, categories, and articles.
- Minimal, text-first design inspired by Airbnb help center.
- WordPress on Kinsta is the preferred CMS.
- No marketing or thought leadership content; focus on help.
- SEO-friendly structure and metadata readiness.

## Current state (repo)
- No WordPress or help-center code exists in this repo.
- No dedicated support blog routes found in `RG-Frontend/`.

## Missing pieces
- WordPress site setup and hosting on Kinsta.
- Content taxonomy, navigation, and article templates.
- Integration strategy with main site (links, header/footer, SSO if needed).

## Next steps
1. Confirm WordPress on Kinsta setup and domain/subdomain plan.
2. Define information architecture (categories and top-level help topics).
3. Build or configure support blog theme (Airbnb-inspired, text-first).
4. Establish content workflow and SEO metadata standards.
