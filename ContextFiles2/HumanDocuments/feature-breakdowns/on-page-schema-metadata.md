# Feature: On-Page Schema/Meta-Data for All Pages

## Overview
A site-wide initiative to ensure structured data and metadata are applied consistently across pages.

## Current state (repo)
- Schema requirements exist for specific features (Therapists SERPs, Suicide Hotline Pages).
- No centralized metadata system is defined.

## Missing pieces
- Standard schema templates by page type.
- Centralized metadata generation or CMS-driven overrides.
- Validation and QA process.

## Next steps
1. Define schema and metadata requirements per page type.
2. Implement shared utilities for meta tags and JSON-LD injection.
3. Add automated validation checks (Rich Results Test sampling).
