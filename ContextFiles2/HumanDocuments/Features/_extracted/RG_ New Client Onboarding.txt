BRD: New client onboarding
Business Requirements Document (BRD)
Feature Name: New Client OnboardingVersion: 1.0Date: Author: Kunal Singh BhatiStakeholders:Reviewers: DavidPaul
1. Executive Summary
Objective:
To implement a primary account holder & sub-account system that allows a Primary user to manage multiple profiles under a single account while ensuring compliance with legal and regulatory requirements for minors and dependent adults.
Business Problem:
The current onboarding process only supports individual users, preventing minors and dependent adults from accessing care through a guardian/ health care agent. Regulatory restrictions require an authorized person to manage their access to licensed care providers.
Proposed Solution
Introduce Primary Account Holders who can create and manage sub-accounts.
Ensure that Protected Health Information (PHI) remains private to each sub-account.
Implement age-based classification to enforce regulatory compliance to minors.
Allow parents or guardians to schedule telehealth appointments for minors or dependent adults.
Key Benefits
Expands accessibility for families while maintaining compliance.
Enhances the user experience by simplifying account management.
Strengthens regulatory adherence by implementing legal age restrictions.
2. Business Goals & Success Metrics
Goals

Provide seamless account management for families.
Success Metrics

Reduction in support tickets related to onboarding and account management.

❌ Out of Scope

User Stories
USER STORIES: NEW CLIENT ONBOARDING
OVERVIEW:
The onboarding flow enables users to create a base account through email/password or third-party options like Google or Apple. It supports account creation for self, children, family members, or clients. The system adjusts based on user age, ensuring parental or guardian consent when needed. Once verified, users can create and manage personalized profiles, making the platform accessible for individuals, families, and professionals alike.

User Story 1: Account Creation
Title:
As a new user, I want to create a new account by entering my details to sign up for the platform and begin using the services.
Description:
The new user should be able to enter their first name, middle name (optional), last name, email, and phone number to create an account. The user must create and confirm a password that meets the security criteria. Users should be able to opt to sign up through Google or Apple sign-in options. The system should validate the user input in real-time to ensure all required fields are filled, and the email and phone number are formatted correctly before submission.
Preconditions:
The user should have a valid email address.
The platform should be able to handle email validation for uniqueness.
Acceptance Criteria:
Form Fields: The "Create an Account" form should have fields for first name, middle name (optional), last name, email, and phone number (optional).
Email Validation: The Email field should include validation for the correct format (e.g., correct email syntax).
Password Requirements: The user must create a password that meets existing security criteria (HIPAA-compliant).
Show/Hide Password: There should be a "show/hide" option for the password fields.
Terms and Conditions: Users must check the "Terms and Conditions" box to proceed, with a link that opens in a new tab.
Sign-Up Options: Users should have the option to sign up with Google or Apple, redirecting to their authentication pages and returning with details pre-filled if authorized.
Error Handling: If any required field is missing or contains invalid input (e.g., invalid email format), an inline error message should appear.
Security and Backend: Passwords must be hashed and stored securely; duplicate email addresses should not be allowed.
Verification Email: Successful account creation should trigger an email verification request Via OTP.
User Story 2: Email Verification
Title:
As a user, I want to verify my email address after signing up to securely proceed to profile creation.
Description:
After submitting the sign-up form, users should verify their email by entering a code sent to their registered email. This process ensures that only valid users can proceed.
Preconditions:
The system should have sent a unique verification code to the user’s email during account creation.
The code should have an expiration period (e.g., 5 minutes). - Not managed on our end
The code will not be sent if a user is onboarded using signup with google or apple.
Acceptance Criteria:
Email Sending: After account creation, an email should contain a one-time verification code (OTP) with a time limit.
Verification Form: A form for the verification code should include fields for input, "Verify," and "Resend code."
Resend Code: The "Resend code" button should become clickable after 30 seconds.
Verification Process: If the code matches, the email is marked as verified. If not, an error is displayed, and "Resend code" is available.
Resend Code: Users can resend the code after a designated period, triggering a new OTP.

Preconditions:
The user has multiple profiles and is performing an action (e.g., booking) after selecting a profile.
Acceptance Criteria:
Any actions like booking or assessment are associated only with the currently active profile.
Profile data from one profile should not overlap with another when switching.
Bookings and other data entries are stored and displayed under the correct profile only.
