# Documentation tactics

- Write short, link-heavy docs; avoid duplicating source content.
- Use imperative checklists for workflows and setup steps.
- Keep scope tight: one skill per reusable workflow.
- Prefer examples that are generic and portable.
- End each doc with a “Hints / next steps” line for follow-up actions.

Hints / next steps: If a doc grows, split it into a skill and an index.
