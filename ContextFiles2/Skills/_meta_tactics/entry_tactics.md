# Entry tactics

- Start with repo instructions (`AGENTS.md`, `CLAUDE.md`, etc.).
- Identify true entry points (server bootstrap, CLI main, UI root) before deep dives.
- Prefer executable projects over libraries, tests, or samples.
- Map dependencies from entry points to core modules to understand the spine.
- Park legacy or demo folders until the main path is confirmed.

Hints / next steps: If entry points are unclear, search for build/run scripts or framework boot files and confirm via references.
