# Skill: testing-guidelines

Description: Use this to standardize testing expectations for contributors.

## Instructions
- Identify the test runner(s) and where tests live in the repo.
- Note naming patterns for tests and any required fixtures or seed data.
- Record how to run full vs. targeted test runs.
- Capture known gaps (missing suites, no coverage targets).

Hints / next steps: Add a minimal smoke test if the repo lacks tests.
