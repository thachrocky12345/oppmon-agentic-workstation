# Meta tactics

Self-audit checklist:
- Coverage: entry, structure, language, data flow, noise filtering, documentation, error avoidance.
- Portability: remove project-specific names and domain terms.
- Clarity: short, imperative bullets; avoid long paragraphs.
- Consistency: each file ends with “Hints / next steps”.
- Evidence: claims in skills map to files or commands where possible.

Hints / next steps: Re-run this checklist after any major repo update.
