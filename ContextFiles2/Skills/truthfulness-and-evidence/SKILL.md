# Skill: truthfulness-and-evidence

Description: Use this to avoid speculation and keep docs trustworthy.

## Instructions
- Link claims to files or commands that back them up.
- Avoid guessing; if unsure, note uncertainty and add a follow-up task.
- Keep evidence local to the repo (paths, commands, logs).
- Validate before declaring “done.”

Hints / next steps: Add a short verification checklist to new docs or runbooks.
