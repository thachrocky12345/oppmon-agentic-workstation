# User Research - Arkon Platform

This directory contains user research artifacts including friction capture templates, observation sessions, and synthesis documents.

## Contents

| File | Purpose |
|------|---------|
| `friction-template.md` | Template for documenting friction points |
| `session-*.md` | Individual observation session notes |
| `synthesis.md` | Aggregated findings and priorities |

## Severity Definitions

### Critical
- User is completely blocked, cannot proceed
- Would require support intervention
- Examples: Login fails with cryptic error, command not found after installation

### Major
- User significantly delayed (>1 minute)
- User expresses frustration or almost gives up
- Examples: Confusing error message, unclear next steps

### Minor
- Brief confusion (<30 seconds)
- User recovers quickly
- Examples: Typo in command, unclear button label

## Observation Protocol

1. Complete onboarding from fresh state
2. Document each friction point with timestamp
3. Note severity, verbalization, and recovery
4. Do not fix during observation - just document
5. Prioritize findings for fix implementation
