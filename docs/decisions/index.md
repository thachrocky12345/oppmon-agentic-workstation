## What is an ADR?

An Architecture Decision Record captures an important architectural decision made along with its context and consequences. They help us:

- Remember why we made certain decisions
- Onboard new team members quickly
- Evaluate decisions when circumstances change

## Status Definitions

- **Proposed**: Under discussion, not yet accepted
- **Accepted**: Approved and in effect
- **Deprecated**: No longer relevant or recommended
- **Superseded**: Replaced by a newer ADR

## Creating a New ADR

1. Copy `_template.md` to `ADR-XXXX-short-title.md`
2. Fill in all sections
3. Submit for review
4. Update this index when accepted

## Auto-Generated ADRs

Some ADRs are auto-generated by the `/init` skill when major dependency changes are detected:

- New frameworks or major libraries added
- Database driver changes
- Authentication library changes
- Infrastructure changes (Docker, K8s)

These auto-generated ADRs are marked with `[AUTO]` in the title and should be reviewed and refined by the team.
